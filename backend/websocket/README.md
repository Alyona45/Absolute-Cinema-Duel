# WebSocket — протокол событий

## Подключение

```
ws://<host>/ws/{room_id}/{user_id}?token=<JWT_ACCESS_TOKEN>
```

- `room_id` — ID комнаты (получается из `POST /rooms` или `POST /rooms/{id}/join`)
- `user_id` — ID участника в комнате (получается из тех же эндпоинтов)
- `token` — JWT access-токен (**обязателен для зарегистрированных**, не нужен для гостей)

### Коды закрытия

| Код  | Причина                                         |
|------|--------------------------------------------------|
| 4001 | Невалидный или отсутствующий JWT (для не-гостей) |
| 4004 | Комната или участник не найдены                  |

---

## HTTP эндпоинты

### Авторизованные (требуют `Authorization: Bearer <token>`)

| Метод | URL                       | Описание                          |
|-------|---------------------------|-----------------------------------|
| POST  | `/rooms`                  | Создать комнату (хост = текущий)  |
| POST  | `/rooms/{room_id}/join`   | Присоединиться к комнате          |

### Гостевые (без авторизации)

| Метод | URL                             | Body                        | Описание                         |
|-------|---------------------------------|-----------------------------|----------------------------------|
| POST  | `/rooms/guest`                  | `{"username": "Вася"}`      | Создать комнату как гость        |
| POST  | `/rooms/{room_id}/join/guest`   | `{"username": "Петя"}`      | Присоединиться как гость         |

> `username` опционален — если не указать, сгенерируется `Гость_abc123`

### Общие

| Метод | URL                       | Описание                          |
|-------|---------------------------|-----------------------------------|
| POST  | `/rooms/{room_id}/start`  | Запустить игру (хост, по user_id) |
| GET   | `/rooms/{room_id}`        | Получить состояние комнаты        |

## События: клиент → сервер

| type   | payload | Описание           |
|--------|---------|---------------------|
| `PING` | —       | Проверка соединения |

---

## События: сервер → клиент

| type                  | payload                                        | Когда                                    |
|-----------------------|------------------------------------------------|------------------------------------------|
| `ROOM_STATE`          | `{host_user_id, participants, status}`         | При подключении — полное состояние       |
| `PLAYER_JOINED`       | `{user_id, username}`                          | Новый участник подключился               |
| `PLAYER_DISCONNECTED` | `{user_id}`                                    | Участник отключился                      |
| `GAME_STARTED`        | `{room_id}`                                    | Хост запустил игру                       |
| `PONG`                | —                                              | Ответ на PING                            |
| `ERROR`               | `{message}`                                    | Невалидный JSON или неизвестный тип      |

---

## Жизненный цикл комнаты

```
1. POST /rooms           → room_id, user_id (хост)
2. POST /rooms/{id}/join → user_id (участник)
3. ws://…/ws/{room_id}/{user_id}?token=… — подключение
4. POST /rooms/{id}/start → GAME_STARTED
5. … мини-игры (будут добавлены) …
6. Комната удаляется автоматически через TTL (60 сек без активных соединений)
```

---

## TTL-уборщик

Фоновая задача проверяет комнаты каждые **60 секунд**.
Комната удаляется если:
- Все участники отключены (`connected = false`)
- С последней активности прошло более **60 секунд**
